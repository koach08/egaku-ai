```
【個人開発】Next.js + FastAPI + Supabase で AI画像生成SaaSを作った話
```

**記事構成**:
```markdown
## はじめに
- 何を作ったか（EGAKU AI の簡単な紹介）
- なぜ作ったか

## 技術スタック
- フロントエンド: Next.js 16 + next-intl（i18n）
- バックエンド: FastAPI + Supabase
- 画像生成: Replicate API + fal.ai API（マルチバックエンド）
- 課金: Stripe
- デプロイ: Vercel + Railway

## 工夫した点
- 15以上のモデルを統一APIで扱う設計
- fal.aiとReplicateのフォールバック構成
- PostgRESTを使ったギャラリー機能
- next-intlによる英語/日本語切り替え

## 苦労した点
- ReplicateのセーフティフィルターのFalse Positive対策
- Supabase RLSとPostgRESTのスキーマキャッシュ問題
- カスタムドメインのSSL証明書問題

## 今後の展望
- ユーザー獲得（ディレクトリ登録、Product Hunt等）
- CivitAIモデル対応
- LoRA対応

## おわりに
サービスURL: https://egaku-ai.com
無料で使えるのでぜひ試してみてください。フィードバック歓迎です。
```

- 技術の話が中心＝Qiitaのルールに合致
